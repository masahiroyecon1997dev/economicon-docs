## 計量経済学の「コードの壁」

計量経済学を学ぶとき、多くの人が最初につまずくのは**分析手法そのものではなく、コードを書くこと**です。

RやPythonで回帰分析を実行するには、パッケージのインストール、データの読み込み構文、引数の指定方法など、分析とは直接関係ない知識が大量に必要になります。その結果、「OLSとはどういう推定量か」「固定効果はいつ使うべきか」といった本質的な問いを考える前に、環境構築やエラー対応で時間を使い果たしてしまう、という状況が生まれます。

```
本来学びたいこと: 推定量の性質・モデルの選び方・結果の解釈
実際に費やす時間: パッケージの使い方・構文エラーの修正・環境設定
```

## Economiconが解決する課題

Economiconは「GUIで操作できること」を最優先に設計されています。

| 課題 | Economiconのアプローチ |
|---|---|
| コードを書けない・書きたくない | すべての操作をマウスクリックで完結 |
| 分析結果の読み方がわからない | 係数・標準誤差・t値・p値・信頼区間をラベル付きで表示 |
| 論文に結果を貼るのが手間 | Markdown・LaTeX形式でワンクリックコピー |
| 手法を試してみたいだけ | インストールしてすぐ使える。設定なし |

## Economiconの位置づけ

EconomiconはRやStataの**代替ではありません**。

本格的な実証研究や再現可能なパイプラインにはコードベースのツールが適しています。Economiconが狙うのはその**手前の段階**——「このデータに回帰をかけるとどうなるか」「この変数を固定効果で制御したら係数はどう変わるか」を素早く確かめたいときです。

```
学習フロー（例）
  Economicon でデータを触り、手法を直感的に理解する
      ↓
  R / Stata / Python で同じ分析を再現し、コードを学ぶ
      ↓
  論文・レポートにまとめる
```

Economiconは「計量経済学への入り口」として、最初の一歩を低コストで踏み出せるようにするためのツールです。

## 誰のためのツールか

- **計量経済学を授業で習ったが、自分で分析したことがない学部生**
- **卒論でデータ分析が必要になったが、RやPythonに慣れていない学生**
- **手法の動作確認や探索的分析をすばやく行いたい研究者・実務家**
